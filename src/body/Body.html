StoneWall Javascript MVVM Framework

Write less code, and keep in control. Stonewall is framework for creating large scalable single page applications. This framework includes features such as - Reusable Components, Routing, Data Binding to DOM Elements, UI auto updates and much more.

Prerequisites

Before you start, you need some knowledge about Requirejs, Web Components, HTML5 custom Element, shadow DOM Concept. For Build environment you need basic knowledge about Nodejs, NPM, Bower and Grunt

This framework contains everything, you need. There is tools and extensions, which ones will give to your application structure.

Maintainable code works for many years. it is - Intuitive, Understandable, Adaptable, Extendable, Debuggable, Testable.

App Structure

Application has two main classes, App and Constructor (Widget). App class will run application, and store context. Also creates EventBus, Router, and models. App handling all communication with Server. App also has access to Container(DOM Element). Important part is, in any Application only one

Constructor (Widget) Class handling all DOM activities. For widgets are no limitations, they can be as many you want, and nested. Each Wdget has to be at least one DOM Element, or more. Any Widget can contain helpers, such as Styles, Components, Placeholders, Bindings and Routes. Below is detailed information about these components.

Styles

Style: In each template has Less css support. Any <style></style> tag supports any css or less. @import tags also supported, just need to use code @{import-url} before. Full example ><style> @import "@{import-url}/_path_to_css.css"; </style>. Full info about Less there. The key feature in templates is, your css are converted to unique classes, which ones applies only to your component. in this way you ne need to worrying about repeatable css class names. This approach is similar to HTML5 Shadow DOM.

Components

Components is AMD modules, means you can define in templates dynamic components. <cp-cpname src="pathtoCMP/Component" ></cp-cpname> Where Component is AMD Javascript file. If Javascript file not necessary there is helper requirejs plugin widget/parser!pathToComponent in this case component can be plain javascript file.

Placeholders

Placeholders are elements which are define location in template. In template you use <pl-name></pl-name> There is two usages for placeholders. First: In Constructor you have access to this element by name.
Second: You can define custom content for this placeholder from other templates.
For example <cp-cpname src="pathtoCMP/Component" ><pl-name>My custom content</pl-name></cp-cpname>

Bindings

Bindings Are used for bind Object data to dom element. in template you will use <bd-name tp-bind="bindName" ></bd-name>. Bindings can be nested, also you can bind attributes as well tp-bind-attrname="dataName". Bindings also support automatic object updates. In this case you need to add attribute tp-update="true"

Routing

Routing are used to handle urls. Routing use UrlManager to make single page applications easy to work with urls. Usage for routes <rt-rtname route="/home" ><div>My custom content</div></rt-rtname>
